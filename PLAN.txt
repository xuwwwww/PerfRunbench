七、Scheduler 設計

每個 request 定義：

[
r_i = (a_i, d_i, x_i)
]

其中：

[
a_i
]

是 arrival time

[
d_i
]

是 deadline

[
x_i
]

是 input size 或 request type

完成時間：

[
f_i
]

latency：

[
L_i = f_i - a_i
]

deadline miss：

[
miss_i = 1[f_i  d_i]
]

要比較的 scheduler：

1. FCFS

request 來了就照順序處理

優點：簡單
缺點：throughput 差，batch utilization 低

2. Static Batching

固定湊 batch size

優點：throughput 比較好
缺點：可能讓早來的 request 等太久

3. Dynamic Batching

設定 max wait time

如果 batch 滿了就跑
如果等太久也跑

優點：平衡 latency 跟 throughput
缺點：沒有 deadline awareness

4. Deadline-Aware Dynamic Batching

根據最早 deadline 決定要不要繼續等

核心規則：

如果繼續等待會讓某個 request 超過 deadline，就立刻 dispatch

這是你自己的設計重點
